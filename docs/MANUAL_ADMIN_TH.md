# คู่มือผู้ดูแลระบบ (Admin) - Tracking System PSUIC

อัปเดตล่าสุด: 3 มีนาคม 2026  
ระบบ: Tracking System PSUIC  
URL ใช้งานจริง: `https://cmdt-uic.psu.ac.th/app/`

## 1) ขอบเขตหน้าที่ของ Admin

บัญชี `admin` ใช้สำหรับ:

1. จัดการผู้ใช้งานและบทบาท
2. จัดการข้อมูลหลักของระบบ (ห้อง, หมวดหมู่, อุปกรณ์)
3. พิมพ์/ส่งออก QR Code อุปกรณ์
4. ดูและส่งออกรายงาน
5. ตั้งค่าสิทธิ์ในหน้า Permission
6. จัดการ Knowledge Base

## 2) เมนูหลักของ Admin

เส้นทางหลัก:

1. `Home` (`/admin`)
2. `Users` (`/admin/manage-users`)
3. `Manage Rooms` (`/admin/manage-rooms`)
4. `Manage Categories` (`/admin/manage-categories`)
5. `Manage Equipment` (`/admin/manage-equipment`)
6. `Reports` (`/admin/reports`)
7. `Permission` (`/admin/permission`)
8. `Knowledge Base` (`/admin/quick-fix`)
9. `Profile` (`/admin/profile`)

## 3) จัดการผู้ใช้งาน (Users)

## 3.1 เพิ่มผู้ใช้

1. เข้า `Users`
2. กด `Add User`
3. กรอก `Full Name`, `Email`, `Role`
4. กำหนดรหัสผ่านสำหรับบัญชีที่ใช้ Email login
5. กด `Create User`

## 3.2 แก้ไข role ผู้ใช้

1. กดแก้ไขในรายการผู้ใช้
2. เลือก role ใหม่ (`user`, `it_support`, `admin`)
3. กดยืนยัน

## 3.3 ลบผู้ใช้

1. กดไอคอนถังขยะ
2. ยืนยันการลบ
3. ระบบไม่อนุญาตให้ลบบัญชีของตัวเอง

## 4) จัดการห้อง (Manage Rooms)

## 4.1 เพิ่ม/แก้ไขห้อง

1. เข้า `Manage Rooms`
2. กด `Add Room` หรือปุ่มแก้ไข
3. กรอก `Room Number`, `Building`, `Floor`
4. บันทึก

## 4.2 ลบห้อง

ระบบมีการป้องกันข้อมูล:

1. ถ้าห้องยังถูกอ้างอิงโดยอุปกรณ์หรือ Ticket จะลบไม่ได้
2. ระบบจะแจ้งจำนวน dependency ที่ยังค้าง
3. ต้องย้าย/ลบข้อมูลที่อ้างอิงห้องนั้นก่อน

## 5) จัดการหมวดหมู่ (Manage Categories)

1. เพิ่มหมวดหมู่หลัก
2. แก้ไขชื่อหมวด
3. ลบหมวดที่ไม่ใช้
4. จัดการ `Sub-Component` ของหมวด (ใช้ในฟอร์มแจ้งซ่อม)

## 6) จัดการอุปกรณ์ (Manage Equipment)

## 6.1 เพิ่มอุปกรณ์

1. กด `Add Equipment`
2. กรอก `Equipment Name`
3. เลือก `Category / Type`
4. เลือก `Room / Location`
5. กรอก `Serial Number` (ถ้ามี)
6. กด `Create Equipment`

เงื่อนไขสำคัญ:

1. ต้องเลือกห้องที่มีอยู่จริง
2. `Serial Number` ต้องไม่ซ้ำกันทั้งระบบ
3. ถ้า serial ซ้ำ ระบบจะไม่สร้างและแจ้ง error

## 6.2 แก้ไขอุปกรณ์

1. กดไอคอนแก้ไข
2. ปรับข้อมูลที่ต้องการ
3. กดบันทึก

## 6.3 ลบอุปกรณ์

1. กดไอคอนถังขยะ
2. ยืนยันการลบ
3. อุปกรณ์ที่ยังมี Ticket active จะลบไม่ได้

## 6.4 ลบหลายรายการ (Bulk Delete)

1. เปิดโหมดเลือกหลายรายการ
2. เลือกรายการที่ต้องการลบ
3. กด `Delete Selected`
4. ระบบจะข้ามรายการที่ยังมี Ticket active และแจ้งรายการที่ลบไม่ได้

## 7) การพิมพ์/ส่งออก QR Code

## 7.1 พิมพ์ QR รายการเดียว

1. เปิด modal QR ของอุปกรณ์
2. กด `Print QR Code`
3. ระบบเปิดหน้า preview สำหรับพิมพ์/บันทึก PDF
4. กด `Print / Save PDF` ในหน้า preview

## 7.2 พิมพ์ QR หลายรายการ

1. เลือกอุปกรณ์หลายรายการ
2. กด `Print Selected QR` หรือ `Print Filtered QR`
3. ระบบจำกัดไม่เกิน 200 รายการต่อรอบ
4. ระบบเรียงลำดับแบบ natural ตามห้อง/อาคาร/ชื่อ/serial เพื่อให้อ่านง่าย

## 7.3 รูปแบบไฟล์เมื่อบันทึก PDF

ระบบตั้งชื่อไฟล์อัตโนมัติ เช่น:

1. `PSUIC-Equipment-QR-single-...`
2. `PSUIC-Equipment-QR-selected-...`
3. `PSUIC-Equipment-QR-filtered-...`

หมายเหตุ:

1. ในเอกสาร QR export เวอร์ชันล่าสุดไม่แสดงลิงก์เว็บไซต์ยาวใต้ QR แล้ว
2. จะคงข้อมูลที่จำเป็น: ชื่ออุปกรณ์, ห้อง, อาคาร, Serial

## 8) รายงานและการส่งออก (Reports)

1. เข้า `Reports`
2. เลือกช่วงเวลา
3. ดูข้อมูลสรุปตามหมวดรายงาน
4. กด `Export PDF` หรือ `Export Excel`

## 9) Permission Management

หน้า `Permission` ใช้กำหนดสิทธิ์ระดับ role ในระบบ เช่น:

1. สิทธิ์การดูงาน
2. สิทธิ์การจัดการข้อมูล
3. สิทธิ์การบริหารผู้ใช้

หมายเหตุ:

1. ฝั่ง API ยังมี role middleware (`adminCheck`, `itCheck`) เป็นด่านหลัก
2. หน้า Permission ใช้เพื่อกำกับ policy และรองรับการขยายสิทธิ์ละเอียดในอนาคต

## 10) Knowledge Base

1. เพิ่มบทความ Quick Fix
2. แก้ไขเนื้อหาเดิม
3. ลบบทความที่ไม่ใช้
4. ใช้เพื่อช่วย user แก้ปัญหาเบื้องต้นก่อนเปิด Ticket

## 11) เช็กลิสต์ Admin รายวัน/รายสัปดาห์

1. ตรวจผู้ใช้ใหม่และ role ให้ถูกต้อง
2. ตรวจความครบถ้วนของ master data (room/category/equipment)
3. ตรวจรายการที่ลบไม่ได้ว่าติด dependency ใด
4. ตรวจรายงานงานค้างและงานเร่งด่วน
5. ส่งออกรายงานเก็บหลักฐานตามรอบองค์กร

## 12) ข้อควรระวังด้านข้อมูล

1. อย่าลบห้อง/อุปกรณ์โดยไม่ตรวจ dependency
2. อย่าสร้างอุปกรณ์ด้วย serial ซ้ำ
3. ไม่แชร์บัญชี admin ร่วมกัน
4. การเปลี่ยนแปลงสำคัญควรบันทึกวันเวลาและผู้ดำเนินการ
