# Executive Handover Summary - Tracking System PSUIC

วันที่จัดทำ: 2 มีนาคม 2026  
ระบบ: Tracking System PSUIC  
URL ใช้งาน: `https://cmdt-uic.psu.ac.th/app/`

---

## 1) ภาพรวมสำหรับผู้บริหาร

ระบบพร้อมใช้งานในงานหลักแล้ว โดยรองรับ 3 บทบาทครบ:

1. `user` แจ้งปัญหาและติดตามงาน
2. `it_support` รับงานและอัปเดตสถานะ
3. `admin` จัดการผู้ใช้ สิทธิ์ ข้อมูลอุปกรณ์ และรายงาน

สถานะปัจจุบัน:

1. Email Login ใช้งานได้จริงใน production
2. PSU Passport ยังไม่เปิดใช้งาน (รออนุมัติจากศูนย์คอมมหาวิทยาลัย + ต้องพัฒนา OAuth callback เพิ่ม)
3. ระบบรันผ่าน PM2 และเสิร์ฟผ่าน Nginx ที่ path `/app/`

---

## 2) สิ่งที่ส่งมอบให้ IT ขององค์กรแล้ว

1. โค้ดระบบ backend + frontend พร้อม deploy runbook
2. เอกสารปฏิบัติการฉบับเต็ม (architecture, SOP, incident/rollback, PSU Passport checklist)
3. แนวทางสำรองข้อมูลและตรวจสุขภาพระบบ
4. คู่มือใช้งานสำหรับ 3 บทบาท (user/it/admin)

เอกสารฉบับเต็ม: `docs/HANDOVER_IT_AND_3ROLES_GUIDE_TH.md`

---

## 3) ความเสี่ยงสำคัญที่ผู้บริหารควรทราบ

### ความเสี่ยงที่ 1: ใช้ฐานข้อมูลร่วมกับระบบอื่น

1. ปัจจุบันระบบนี้เก็บข้อมูลใน DB ชื่อ `wordpress` ร่วมกับระบบอื่นขององค์กร
2. ความเสี่ยงคือการปฏิบัติการผิดพลาดอาจส่งผลกระทบข้ามระบบ

แนวทางควบคุม:

1. ควบคุมคำสั่ง production ด้วย SOP กลาง
2. สำรองข้อมูลก่อน migration ทุกครั้ง
3. วางแผนแยก DB เฉพาะระบบนี้ในระยะถัดไป

### ความเสี่ยงที่ 2: PSU Passport ยังไม่พร้อมเปิดจริง

1. มีผลให้ผู้ใช้งานต้องใช้ Email Login ชั่วคราว
2. ไม่มีผลต่อความสามารถหลักของระบบติดตามงาน

แนวทางควบคุม:

1. ดำเนินงาน cutover ตาม checklist เมื่อได้รับอนุมัติ
2. ทดสอบ UAT ก่อนเปิดจริง
3. มี rollback flag ปิด PSU Login ได้ทันทีหากเกิดปัญหา

---

## 4) สิ่งที่ต้องการการตัดสินใจจากผู้บริหาร

1. อนุมัติให้ IT ใช้ SOP production อย่างเคร่งครัด (โดยเฉพาะห้ามรันคำสั่ง seed บน production)
2. อนุมัติโครงการย่อย "PSU Passport Integration" หลังได้รับเอกสารอนุมัติจากศูนย์คอมฯ
3. อนุมัติ roadmap แยกฐานข้อมูลระบบนี้ออกจาก DB `wordpress`

---

## 5) Operational Ownership หลังส่งมอบ

1. เจ้าของระบบเชิงปฏิบัติการ: ทีม IT องค์กร
2. เจ้าของระบบเชิงนโยบายสิทธิ์: ผู้ดูแลระบบที่องค์กรแต่งตั้ง
3. จุดควบคุมเปลี่ยนแปลง: deploy ผ่าน runbook ที่กำหนดเท่านั้น

---

## 6) KPI ที่แนะนำให้ติดตามรายเดือน

1. อัตรา ticket ที่ปิดงานสำเร็จ
2. เวลาตอบรับงานเฉลี่ย (response time)
3. เวลาแก้ไขงานเฉลี่ย (resolution time)
4. จำนวน incident production
5. ความพร้อมระบบ (uptime)

---

## 7) ข้อความรับรองสถานะส่งมอบ (พร้อมใช้)

ระบบ `https://cmdt-uic.psu.ac.th/app/` อยู่ในสถานะ "พร้อมใช้งานเชิงปฏิบัติการ" สำหรับ Email Login และการทำงานหลักทั้ง 3 บทบาท โดยมีข้อจำกัดเชิงแผนงานคือ PSU Passport ยังอยู่ระหว่างรออนุมัติและดำเนินการ cutover.
