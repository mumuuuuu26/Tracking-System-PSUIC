# คู่มือเจ้าหน้าที่ IT Support - Tracking System PSUIC

อัปเดตล่าสุด: 3 มีนาคม 2026  
ระบบ: Tracking System PSUIC  
URL ใช้งานจริง: `https://cmdt-uic.psu.ac.th/app/`

## 1) บทบาท IT ในระบบ

บัญชี `it_support` ใช้สำหรับ:

1. รับงานที่ user แจ้งเข้ามา
2. ปฏิเสธงานพร้อมระบุเหตุผล (เมื่อไม่เข้าเงื่อนไข)
3. อัปเดตความคืบหน้าและบันทึกงานแก้ไข
4. ปิดงานพร้อมหลักฐาน
5. ตั้งค่าแจ้งเตือนอีเมลส่วนตัว
6. เชื่อม Google Calendar เพื่อแสดงตารางงาน

## 2) เริ่มใช้งานครั้งแรก

1. เข้า `https://cmdt-uic.psu.ac.th/app/login`
2. กด `Login with Email`
3. กรอกบัญชี `it_support`
4. เข้าสู่หน้า IT dashboard (`/it`)

หมายเหตุ:

1. PSU Passport ยังไม่เปิดใช้งานจริงในระบบนี้ ณ วันที่ 3 มีนาคม 2026
2. ให้ใช้ Email/Password เท่านั้น

## 3) ตั้งค่าโปรไฟล์และการแจ้งเตือนอีเมล

### 3.1 ตั้งค่าพื้นฐานโปรไฟล์

1. เข้าเมนู `Profile` (`/it/profile`)
2. อัปเดตชื่อ, เบอร์โทร, แผนก, เบอร์ภายใน, เวลาทำงาน
3. กดบันทึกแต่ละส่วน

### 3.2 ตั้งค่า Email Notification

1. อยู่หน้า `Profile`
2. เปิดสวิตช์แจ้งเตือนอีเมล (`Enable/Disable`)
3. ใส่ `Notification Email` ที่ต้องการรับแจ้งเตือน
4. กด `Save` หรือบันทึกผ่านปุ่มอัปเดต

เงื่อนไขที่ต้องพร้อม:

1. ฝั่งเซิร์ฟเวอร์ต้องมี `MAIL_USER` และ `MAIL_PASS` ใน `.env.production`
2. ถ้าเซิร์ฟเวอร์ยังไม่ตั้ง SMTP คุณยังบันทึกอีเมล/สวิตช์ได้ แต่ระบบจะยังไม่ส่งเมลจนกว่า admin จะตั้งค่าเสร็จ

## 4) ตั้งค่า Google Calendar

### 4.1 เปิดหน้า Schedule

1. เข้าเมนู `Schedule` (`/it/schedule`)
2. กดไอคอนผู้ใช้มุมขวาบนเพื่อเปิด `Connection Setup`

### 4.2 Step 1: Share Calendar ให้ Service Account

1. คัดลอก `Service Email` ที่ระบบแสดง
2. ไปที่ Google Calendar ของคุณ
3. เปิดสิทธิ์แชร์ปฏิทินให้ Service Email (อย่างน้อยสิทธิ์อ่านกิจกรรม)

### 4.3 Step 2: กรอก Calendar ID

1. นำ `Calendar ID` มากรอกในหน้า setup
2. กด `Save & Connect` (หรือ `Save Calendar ID` หากฝั่งเซิร์ฟเวอร์ยังไม่พร้อม)
3. ถ้า key Google พร้อม ระบบจะ sync ปฏิทินเข้าระบบทันที
4. ถ้า key ยังไม่ครบ ระบบจะบันทึก Calendar ID ไว้ก่อน และเริ่ม sync อัตโนมัติหลัง admin ตั้งค่า server ครบ

ข้อสังเกต:

1. ระบบมี cooldown การ sync เพื่อลดภาระ (`GOOGLE_SYNC_MIN_INTERVAL_MS`, ค่าเริ่มต้น 5 นาที)
2. หาก key ฝั่ง Google ในเซิร์ฟเวอร์ไม่ครบ ระบบจะแจ้งรายการ key ที่ขาด

## 5) ขั้นตอนทำงานกับ Ticket (Workflow หลัก)

## 5.1 รับงานจาก Dashboard

1. เข้า `Dashboard` (`/it`)
2. เลือกงานที่สถานะ `Not Start`
3. กด `Accept` เพื่อรับงาน
4. หากต้องปฏิเสธงาน กด `Reject` และระบุเหตุผล

## 5.2 อัปเดตงานในหน้า Ticket Detail

1. เข้า `/it/ticket/:id`
2. ตรวจรายละเอียดปัญหา, ห้อง, หมวด, รูปก่อนซ่อม
3. เพิ่ม `Checklist` งานย่อย
4. กรอก `IT Notes` ระหว่างแก้ไข
5. แนบรูปหลักฐานหลังแก้ไข (ถ้ามี)

## 5.3 บันทึกฉบับร่าง

1. เมื่อสถานะเป็น `in_progress` กด `Save Draft`
2. ระบบจะเก็บ note/checklist/รูปหลักฐานที่กรอกไว้

## 5.4 ปิดงาน

1. เปลี่ยนสถานะเป็น `completed`
2. ตรวจความครบถ้วนของ checklist
3. กดยืนยัน `Complete Job`
4. ระบบจะบันทึก `completedAt` และคำนวณ `resolutionTime`

## 5.5 กฎสำคัญของสถานะ

1. `not_start` -> รับงานแล้วเป็น `in_progress`
2. `in_progress` -> ปิดงานเป็น `completed`
3. `rejected` และ `completed` เป็นสถานะปลายทาง (แก้ไขต่อไม่ได้)
4. ระบบไม่รองรับการย้อนกลับไป `not_start` ด้วยการแก้มือ

## 6) เมนูเสริมที่ IT ใช้บ่อย

1. `Tickets` (`/it/tickets`) - ค้นหาและกรองงานทั้งหมด
2. `History` (`/it/history`) - ดูงานที่ปิดโดย IT คนนั้น
3. `Notifications` (`/it/notifications`) - อ่านแจ้งเตือนและกดเปิด ticket ตรง

## 7) Troubleshooting สำหรับ IT

1. รับงานไม่ได้และขึ้นว่า assigned to another IT
แนวทาง: ให้ตรวจว่า ticket ถูก assign แล้วโดยใคร

2. ปิดงานไม่ได้
แนวทาง: ตรวจว่ากรอก note แล้วและ ticket ไม่อยู่สถานะ rejected/completed ไปแล้ว

3. sync ปฏิทินไม่เข้า
แนวทาง: ตรวจสิทธิ์ share calendar, ตรวจ Calendar ID, ตรวจ Google key ฝั่งเซิร์ฟเวอร์

4. ไม่ได้รับอีเมลแจ้งเตือน
แนวทาง: ตรวจว่าเปิด isEmailEnabled, notificationEmail ถูกต้อง, และ SMTP server พร้อม

## 8) เช็กลิสต์ปฏิบัติการรายวันสำหรับ IT

1. เปิด Dashboard ตรวจคิวงานใหม่
2. รับงานที่เร่งด่วน (`High`) ก่อน
3. อัปเดต note/checklist ให้ครบทุกเคสที่กำลังทำ
4. ปิดงานที่เสร็จแล้วทันที ไม่ปล่อยค้าง
5. ตรวจ Notifications เผื่องานที่ถูก mention หรืออัปเดต
6. Sync ปฏิทินเมื่อมีการเปลี่ยนตารางงานจริง

## 9) ข้อควรระวัง

1. ไม่ควรปิดงานโดยไม่ใส่รายละเอียดการแก้ไข
2. ไม่ควรปฏิเสธงานโดยไม่ระบุเหตุผลชัดเจน
3. ไม่แชร์บัญชี IT ร่วมกันหลายคน
4. หากพบความผิดปกติของข้อมูล ให้แจ้ง admin ทันที
