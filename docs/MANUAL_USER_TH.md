# คู่มือผู้ใช้งานทั่วไป (User) - Tracking System PSUIC

อัปเดตล่าสุด: 3 มีนาคม 2026  
ระบบ: Tracking System PSUIC  
URL ใช้งานจริง: `https://cmdt-uic.psu.ac.th/app/`

## 1) เป้าหมายของคู่มือนี้

คู่มือนี้สำหรับผู้ใช้งานทั่วไปที่ต้องการ:

1. สแกน QR แล้วเข้าฟอร์มแจ้งปัญหาได้ทันที
2. ลงชื่อเข้าใช้และแจ้งปัญหาได้ถูกต้อง
3. ติดตามสถานะงานซ่อมจนจบ
4. ประเมินงานหลังปิดเคสได้

## 2) ก่อนเริ่มใช้งาน

1. เตรียมบัญชีผู้ใช้ระบบ (Email/Password)
2. ใช้ Browser รุ่นใหม่ (Chrome, Safari, Edge)
3. เปิดเว็บที่ `https://cmdt-uic.psu.ac.th/app/`
4. หากจะสแกนจากมือถือ ให้อนุญาตสิทธิ์กล้อง

หมายเหตุสำคัญ:

1. ณ วันที่ 3 มีนาคม 2026 ปุ่ม `PSU Passport` ยังปิดใช้งาน
2. วิธีเข้าใช้งานที่ทำงานได้จริงตอนนี้คือ `Login with Email`

## 3) ขั้นตอนหลัก: สแกน -> ล็อกอิน -> แจ้งปัญหา

### 3.1 สแกน QR จากอุปกรณ์

1. ใช้กล้องมือถือสแกน QR บนอุปกรณ์
2. ลิงก์ที่ถูกต้องจะพาไปรูปแบบ `https://cmdt-uic.psu.ac.th/app/scan/<qr-code>`
3. ถ้ายังไม่ล็อกอิน ระบบจะพาไปหน้า `login` อัตโนมัติ และจำลิงก์ปลายทางไว้ให้
4. Flow นี้ใช้สำหรับบัญชี `user` เท่านั้น หากล็อกอินด้วยบัญชี `it_support` หรือ `admin` ระบบจะแจ้งเตือนและพาไปหน้า dashboard ของ role นั้น

### 3.2 ลงชื่อเข้าใช้

1. ที่หน้า Login ให้กด `Login with Email`
2. กรอก Email และ Password
3. กด `Sign In`
4. ระบบจะพากลับไป flow สแกนเดิม แล้วเปิดฟอร์ม `Report Issue` พร้อมข้อมูลอุปกรณ์ที่อ่านจาก QR

### 3.3 กรอกฟอร์มแจ้งปัญหา (Report Issue)

กรอกข้อมูลที่จำเป็น:

1. `Topic Issue`
2. `Equipment Category`
3. `Describe the Issue`
4. `Floor`
5. `Room`
6. `Priority Level` (`Low`, `Medium`, `High`)

ข้อมูลเสริม:

1. `Sub-Component` (ถ้ามี)
2. รูปภาพประกอบ (ไม่บังคับ)

เมื่อกรอกครบแล้ว:

1. กด `Submit Report`
2. ระบบจะขึ้นข้อความสำเร็จและพาไปหน้ารายละเอียด Ticket

## 4) ทางเลือก: แจ้งปัญหาแบบไม่สแกน QR

1. ล็อกอินเข้าระบบ
2. เข้าเมนู `Report Issue` (`/user/create-ticket`)
3. เลือกหมวด, ชั้น, ห้อง และกรอกรายละเอียดด้วยตนเอง
4. กด `Submit Report`

## 5) ติดตามสถานะงาน

เมนูที่ใช้บ่อย:

1. `Home` - ดูงานล่าสุดและทางลัด
2. `Activity` (`/user/report`) - ดูรายการ Ticket ทั้งหมดพร้อม filter สถานะ/ชั้น/ห้อง
3. `Task History` (`/user/history`) - ดูงานที่ปิดแล้วหรือจบกระบวนการ
4. `Ticket Detail` (`/user/ticket/:id`) - ดู timeline และหลักฐานการแก้ไข

ความหมายสถานะ:

1. `not_start` = ยังไม่มีเจ้าหน้าที่รับงาน
2. `in_progress` = เจ้าหน้าที่รับงานแล้วและกำลังดำเนินการ
3. `completed` = งานเสร็จแล้ว
4. `rejected` = งานถูกปฏิเสธพร้อมเหตุผล

## 6) การประเมินงานหลังปิดเคส

เมื่อ Ticket เป็น `completed`:

1. เข้าเมนู `Activity` หรือ `History`
2. กดปุ่ม `Rating` บนการ์ดงาน

การเปิดฟอร์มประเมินมี 2 กรณี:

1. ถ้าองค์กรตั้งค่า `VITE_EXTERNAL_RATING_FORM_URL` แล้ว ระบบจะเปิดฟอร์มภายนอก
2. ถ้ายังไม่ตั้งค่า ระบบจะพาไปหน้ารายละเอียด Ticket ภายในระบบแทน

## 7) ปัญหาที่พบบ่อยและวิธีแก้

1. สแกนแล้วขึ้นว่า `Equipment was not found`
แก้ไข: ให้สแกนใหม่ หรือแจ้ง IT ตรวจว่า QR ของอุปกรณ์ยังมีในฐานข้อมูล

2. กดส่งฟอร์มแล้วไม่ผ่าน
แก้ไข: ตรวจว่ากรอกครบ `Topic`, `Category`, `Description`, `Room`, `Priority`

3. รูปแนบไม่ขึ้น
แก้ไข: อัปโหลดใหม่และใช้ไฟล์ภาพขนาดไม่ใหญ่เกินไป

4. ปุ่ม PSU Passport กดไม่ได้
แก้ไข: เป็นพฤติกรรมปัจจุบันของระบบ ให้ใช้ `Login with Email` แทน

## 8) ข้อมูลที่ระบบจัดเก็บจากการใช้งาน

ระบบจะเก็บข้อมูลหลักต่อไปนี้:

1. ข้อมูลบัญชีผู้ใช้ (`email`, `name`, บทบาท)
2. ข้อมูล Ticket (`title`, `description`, `urgency`, `status`, ห้อง/อุปกรณ์)
3. รูปภาพประกอบการแจ้งงาน
4. ประวัติการอัปเดตสถานะและการแจ้งเตือน

## 9) ข้อมูลที่ควรส่งให้ IT เมื่อแจ้งปัญหาระบบ

หากระบบใช้งานไม่ได้ ให้ส่งข้อมูลนี้ให้ IT:

1. เวลาเกิดเหตุ (วัน/เวลา)
2. URL ที่เกิดปัญหา
3. Screenshot หน้าจอ
4. ขั้นตอนที่ทำก่อนเกิดปัญหา
5. หมายเลข Ticket (ถ้ามี)
