# คู่มือผู้ใช้งานทั่วไป (User) - Tracking System PSUIC

อัปเดตล่าสุด: 3 มีนาคม 2026  
ระบบ: Tracking System PSUIC  
URL ใช้งานจริง: `https://cmdt-uic.psu.ac.th/app/`

## 1) คู่มือนี้ใช้กับใคร

คู่มือนี้สำหรับผู้ใช้งานบทบาท `user` ที่ต้องการ:

1. สแกน QR อุปกรณ์และแจ้งปัญหาได้ถูกต้อง
2. ลงชื่อเข้าใช้ระบบและส่ง Ticket ได้
3. ติดตามสถานะงานจนจบ
4. ประเมินงานหลังซ่อมเสร็จ

## 2) สิ่งที่ควรรู้ก่อนเริ่ม

1. ระบบที่ใช้งานจริงตอนนี้ใช้ `Login with Email`
2. ปุ่ม `PSU Passport` ยังไม่เปิดใช้งานจริงใน production (ณ วันที่ 3 มีนาคม 2026)
3. ควรใช้เบราว์เซอร์รุ่นใหม่ เช่น Chrome, Safari, Edge
4. หากใช้มือถือสแกน QR ควรอนุญาตสิทธิ์กล้อง

## 3) วิธีใช้งานหลัก: สแกน -> ล็อกอิน -> แจ้งปัญหา

### 3.1 สแกน QR จากอุปกรณ์

1. เปิดกล้องมือถือแล้วสแกน QR ที่ติดกับอุปกรณ์
2. ลิงก์ที่ถูกต้องจะเปิดประมาณ `https://cmdt-uic.psu.ac.th/app/scan/<qr-code>`
3. ถ้ายังไม่ล็อกอิน ระบบจะพาไปหน้า `/login` อัตโนมัติ
4. หลังล็อกอินสำเร็จ ระบบจะพากลับ flow สแกนเดิมอัตโนมัติ

หมายเหตุ:

1. Flow สแกนนี้ออกแบบมาสำหรับ role `user`
2. ถ้าบัญชี `it_support` หรือ `admin` เปิดลิงก์สแกน ระบบจะแจ้งเตือนและพาไปหน้า role ของตน

### 3.2 เข้าสู่ระบบ

1. เข้า `https://cmdt-uic.psu.ac.th/app/login`
2. กด `Login with Email`
3. กรอกอีเมลและรหัสผ่าน
4. กด `Sign In`

## 4) วิธีแจ้งปัญหา (Report Issue)

เมื่อเข้าแบบสแกน QR สำเร็จ ระบบจะ prefill ข้อมูลอุปกรณ์บางส่วนให้ทันที

ฟิลด์ที่ต้องกรอก:

1. `Topic Issue`
2. `Equipment Category`
3. `Describe the Issue`
4. `Floor`
5. `Room`
6. `Priority Level` (`Low`, `Medium`, `High`)

ฟิลด์ไม่บังคับ:

1. `Sub-Component` (ถ้ามี)
2. รูปภาพประกอบปัญหา

ขั้นตอนส่งงาน:

1. กรอกข้อมูลให้ครบ
2. กด `Submit`
3. ระบบจะแสดงข้อความสำเร็จ
4. กด `View Ticket` เพื่อเข้าไปหน้ารายละเอียดงาน

## 5) ถ้าไม่ได้สแกน QR แต่ต้องการแจ้งงาน

1. ล็อกอินให้เรียบร้อย
2. เข้าเมนู `Report Issue` หรือ URL `/user/create-ticket`
3. เลือกหมวดหมู่ ชั้น ห้อง และกรอกอาการเสียด้วยตนเอง
4. กด `Submit`

## 6) การติดตามสถานะงาน

เมนูที่ใช้บ่อย:

1. `Activity` (`/user/report`) สำหรับดูรายการ Ticket แบบกรองสถานะ
2. `History` (`/user/history`) สำหรับดูประวัติย้อนหลัง
3. `Ticket Detail` (`/user/ticket/:id`) สำหรับดูรายละเอียดงานแต่ละใบ

ความหมายสถานะ:

1. `not_start` = ยังไม่มีเจ้าหน้าที่รับงาน
2. `in_progress` = เจ้าหน้าที่กำลังดำเนินการ
3. `completed` = งานเสร็จเรียบร้อย
4. `rejected` = งานถูกปฏิเสธพร้อมเหตุผล

## 7) การประเมินงานหลังงานเสร็จ

เมื่อ Ticket อยู่สถานะ `completed`:

1. เข้า `Activity` หรือ `History`
2. กดปุ่ม `Rating`

พฤติกรรมการเปิดแบบประเมิน:

1. หากระบบตั้ง `VITE_EXTERNAL_RATING_FORM_URL` จะเปิดแบบประเมินภายนอก
2. หากยังไม่ตั้งค่า ระบบจะพาไปหน้ารายละเอียด Ticket ในระบบแทน

## 8) ปัญหาที่พบบ่อยและวิธีแก้

1. สแกนแล้วขึ้นว่าไม่พบอุปกรณ์  
แนวทาง: สแกนใหม่และตรวจว่า QR เป็นของระบบนี้จริง หากยังไม่พบให้แจ้ง IT ตรวจฐานข้อมูลอุปกรณ์

2. กดส่ง Ticket ไม่ได้  
แนวทาง: ตรวจว่ากรอกครบ `Topic`, `Category`, `Description`, `Room`, `Priority`

3. แนบรูปแล้วไม่ขึ้น  
แนวทาง: ลองอัปโหลดรูปใหม่ หรือใช้ไฟล์ภาพขนาดเล็กลง

4. เปิดเว็บแล้วหลุดไปหน้า `404 Not Found`  
แนวทาง: กลับไป `https://cmdt-uic.psu.ac.th/app/` แล้วล็อกอินใหม่

## 9) ข้อมูลที่ระบบจัดเก็บจากผู้ใช้

ระบบจัดเก็บข้อมูลที่เกี่ยวกับงานซ่อม เช่น:

1. ข้อมูลบัญชีผู้ใช้ (อีเมล ชื่อ บทบาท)
2. ข้อมูล Ticket (หัวข้อ รายละเอียด ความเร่งด่วน ห้อง หมวดหมู่)
3. รูปภาพประกอบที่ผู้ใช้แนบ
4. ประวัติการเปลี่ยนสถานะงาน

## 10) ข้อมูลที่ควรส่งให้ IT เมื่อแจ้งปัญหาระบบ

1. วันและเวลาที่เกิดปัญหา
2. URL ที่เกิดปัญหา
3. Screenshot หน้าจอ
4. ขั้นตอนที่ทำก่อนเกิดปัญหา
5. หมายเลข Ticket (ถ้ามี)
